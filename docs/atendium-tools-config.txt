CONFIGURACION DE HERRAMIENTAS "CUSTOM API REST CALL" - ATENDIUM
================================================================

Este documento tiene, campo por campo, lo que hay que cargar en el formulario
"Custom API Rest Call" de la plataforma Atendium para cada endpoint de
bausing-backend/routes/atendium.py.

Base URL: https://bausing-api-qy2vc.ondigitalocean.app/atendium/v1


CREDENCIAL (crear una sola vez y reutilizar en las 10 herramientas)
---------------------------------------------------------------
Tipo: API Key por header
Nombre del header: X-API-Key
Valor: el valor de API_KEY configurado en el backend
Nombre sugerido de la credencial: Bausing API Key

Con esto seteado, en cada herramienta de abajo NO hace falta activar el
toggle de "Headers" para la auth: se selecciona directamente en el campo
"Credencial".

IMPORTANTE - "Reintentar ante fallas de la API":
Dejalo APAGADO en "Crear Orden" (POST /orders). Si Atendium reintenta
automaticamente una creacion de orden que si llego a procesarse en el
servidor pero fallo la respuesta (timeout, corte de red), se puede duplicar
la venta en el CRM. En el resto de los endpoints (todos de solo lectura o
sin efectos secundarios persistentes) es seguro dejarlo encendido.

En todas las herramientas: "Enviar Metadata del mensaje a la API body" y
"Enviar la respuesta de la API directamente" -> apagados (dejamos que el
agente redacte la respuesta al usuario final).

================================================================
Titulo de la herramienta: Bausing - Resolver Zona

FLUJO SUGERIDO PARA EL BOT
================================================================
1. "Resolver Zona" con la direccion -> mostrar dias estimados.
2. "Buscar Catalogo" con locality_id de la zona.
3. "Validar Cupon" / "Validar Referido" si el cliente los menciona.
4. "Cotizar Pedido" con el carrito armado.
5. Si requires_human_handoff -> derivar con handoff_message + whatsapp_phone.
6. Si can_create_order -> confirmar datos con el cliente -> "Crear Orden".
7. "Estado de Pedido" / "Listar Pedidos" para seguimiento.


================================================================
TEXTO PARA EL CAMPO "INSTRUCCIONES" DEL BOT (pegar tal cual)
================================================================
Sos el asistente de ventas de Bausing por WhatsApp. Tu objetivo es ayudar al cliente a elegir productos, cotizar y cerrar la compra con pago contra entrega, o derivarlo a un asesor humano cuando corresponda. Se cordial, breve y claro, usa pesos argentinos y nunca inventes precios, stock ni tiempos de entrega: siempre salen de las herramientas.

PASOS OBLIGATORIOS (segui este orden):

1. Pedi la direccion completa del cliente (calle, numero, ciudad, provincia, codigo postal; si tiene ubicacion de WhatsApp, mejor). Usa la herramienta "Bausing - Resolver Zona" [POST /resolve-zone] para obtener localidad, catalogo y dias estimados de entrega. Contale los dias estimados.

2. Pregunta que productos busca y usa "Bausing - Buscar Catalogo" [GET /catalog] (con el locality_id de la zona) para mostrarle opciones con precio. Si pide detalle de un producto puntual, usa "Bausing - Detalle de Producto" [GET /catalog/:product_id].

3. Si el cliente menciona un cupon, validalo con "Bausing - Validar Cupon" [POST /validate-coupon] antes de aplicarlo. Si menciona un codigo de referido, validalo con "Bausing - Validar Referido" [POST /validate-referral].

4. Antes de confirmar cualquier compra, arma el carrito y usa "Bausing - Cotizar Pedido" [POST /quote] con la direccion, los items, el medio de pago y cupon/referido si los hay. Nunca calcules el total vos mismo.

5. Si la cotizacion devuelve requires_human_handoff en true, no sigas con la venta: deriva al cliente al asesor usando el handoff_message y el whatsapp_phone que te da la herramienta.

6. Si can_create_order es true, pregunta y confirma con el cliente: nombre y apellido, DNI, email, telefono, medio de pago (efectivo, transferencia o tarjeta) y que la direccion sea correcta. Con esos datos confirmados, crea la venta con "Bausing - Crear Orden" [POST /orders].

7. Si el cliente pregunta por pedidos anteriores, pedile telefono y/o DNI y usa "Bausing - Listar Pedidos" [GET /orders]. Si pregunta por el estado de un pedido puntual, pedile el numero de pedido y su telefono y usa "Bausing - Estado de Pedido" [GET /orders/:order_id].

PREGUNTAS QUE SIEMPRE DEBES HACER ANTES DE CREAR UNA ORDEN:
- Nombre y apellido
- DNI
- Email
- Telefono de contacto
- Direccion completa (si no la diste antes)
- Medio de pago preferido

QUE NO HACER:
- No inventes precios, stock, cupones ni tiempos de entrega: siempre verificalos con la herramienta correspondiente.
- No crees una orden sin haber cotizado antes con "Bausing - Cotizar Pedido" [POST /quote].
- No crees una orden si la cotizacion pide derivar a un asesor.
- No repitas la cotizacion completa en cada mensaje; resumi lo esencial (productos, total, envio).
